Monta NL – Odoo Integration

Automated order synchronization and fulfilment connection between Monta and Odoo

This module connects your Odoo system with Monta NL (Warehouse and Fulfilment platform) to automate the synchronization of orders, shipping details, and fulfilment statuses. It ensures your logistics process stays connected, accurate, and up to date.

Orders from Monta can be pulled into Odoo, and Odoo orders (including subscription renewals) are automatically pushed to Monta, removing the need for manual data entry.

Key Features

  • Automatic synchronization of orders between Monta and Odoo.
  • Automatic creation of Monta orders when Odoo subscription orders renew.
  • Prevention of duplicate orders using Monta order references.
  • Sync of Monta order status, tracking numbers, and fulfilment details into Odoo.
  • Manual and scheduled synchronization options.
  • Dedicated configuration panel inside Odoo.
  • Detailed and secure error logging.

Configuration

The module includes its own configuration menu where all Monta connection and access settings can be managed easily from the Odoo interface.

Navigate to:
Monta → Configuration → Monta Configuration

From there, you can configure:

  • Monta API Base URL
  • API Username and Password
  • Request timeout
  • Allowed Odoo base URLs
  • Allowed companies for Monta operations
  • Security origin settings

All settings are managed from a structured form view, making setup simple, secure, and fully controlled from within Odoo — no technical configuration or system parameters are required.

Subscription Order Automation

This module supports automated fulfilment for Odoo Subscriptions. Whenever a subscription renews and generates a new sales order, the order is automatically sent to Monta without any manual action.

This allows businesses running recurring shipments to fully automate their warehouse and fulfilment flow.

Security Notes

  • API credentials are securely stored and only accessible to authorized Odoo users.
  • All communication with Monta uses HTTPS.
  • Errors and API failures are logged for troubleshooting without exposing sensitive data.
  • It is recommended to restrict access to the Monta configuration menu to administrators only.

Technical Overview

  • Centralized Monta API client for all requests.
  • Configurable timeouts and secure authentication.
  • Supports automated background jobs and manual triggers.
  • Integrated logging for monitoring and debugging.

Installation Requirements

  • Odoo 18.0
  • Python package requests
  • Active Monta API credentials

Example Workflow

  • Configure Monta credentials from the Monta Configuration menu.
  • Create or sync sales orders in Odoo.
  • Orders are automatically sent to Monta for fulfilment.
  • Monta returns status updates and tracking details.
  • Odoo orders are updated in real time.
